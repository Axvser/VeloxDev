## Skill: 连接验证（关键——防止无限重试）

框架可能**静默拒绝**连接尝试。所有 Connect 工具现在会验证并报告拒绝情况。  
收到 `status: "rejected"` 时，**禁止重试相同的连接**。拒绝是确定性的——相同输入始终被拒绝。

### 拒绝原因（来自响应的 `reasons` 字段）

1. **通道不兼容**：发送方插槽的通道不支持发送，或接收方插槽的通道不支持接收。检查 GetNodeDetail/ListSlotProperties 中的 `ch` 字段。
2. **同节点连接**：发送方和接收方属于同一节点——始终被拒绝。
3. **开发者 ValidateConnection 规则**：Tree 的 helper 包含自定义验证逻辑，对 Agent 不透明。若因此原因被拒绝，请告知用户。
4. **通道容量限制**：单目标/单来源通道可能已满（现有连接占用了唯一槽位）。可能需要先断开现有连接。
5. **同向连接自动去重**：当连接两个已有同向连接的节点时，框架会**静默删除旧连接**。结果是替换而非并行连接。此操作自动发生——对同一节点对，你**无需**先调用 `DisconnectSlots` 再调用 `ConnectSlots`。

### 恢复策略

读取 `reasons` 数组：
- 若是通道问题 → 检查并考虑用 SetSlotChannel 更改通道；
- 若是容量问题 → 先断开现有连接；
- 若是 ValidateConnection → 告知用户。

### 连接工具的刷新策略

- 若前几步**没有**改变拓扑，可继续复用缓存的 slot ID。
- 若最近改动了插槽集合、删除了组件、克隆了节点，或调用了 `SetEnumSlotCollection`，则应先刷新插槽身份信息。
- 在未改变前置条件（channel、端点、selector 类型、容量占用）前，不要对同一 `Connect*` 调用做循环重试。
