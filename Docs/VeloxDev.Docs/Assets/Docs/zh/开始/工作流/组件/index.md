# 组件

工作流是一种图结构，包含四个重要的组件，本章节将为您展示如何构建组件对应的视图模型

> **组件一览**

| 组件类型 | 父级类型 | 描述 |
| --- | --- | --- |
| Tree |  | 全局容器，存储单个工作流的一切数据 |  |
| Node | Tree | 节点，承载配置与业务执行 |  |
| Slot | Node | 连接器，承载连接关系与通道方向限制 |  |
| Link |  | 连接线，可访问始发点与终止点的 Slot |  |

> **核心特性**

接下来，所有的视图模型都将基于标注下述特性去构建，$component是组件类型，$helper是注入的业务逻辑

```csharp
【WorkflowBuilder.{$component}<{$helper}>】
```
